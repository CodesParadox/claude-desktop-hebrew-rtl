

# תיקון עברית (RTL) ל‑Claude Desktop

**תמיכה טבעית בעברית (כיוון מימין‑לשמאל) עבור Claude Desktop ב‑Windows.**

הכלי מזהה אוטומטית טקסט עברי ומיישר את הכיוון במקומות הנכונים — בתיבת הקלט וגם
בתשובות של Claude, כולל בזמן שהתשובה עדיין נכתבת (streaming) — בלי לשבור אנגלית,
קוד, נוסחאות או טבלאות.

> פרויקט זה הוא לעברית בלבד. אין בו תמיכה בערבית או בשפות RTL אחרות, מתוך כוונה
> שהחוויה תהיה מותאמת למשתמשי עברית.

---

## מה הכלי עושה

- **מזהה עברית אוטומטית** בתשובות של Claude ובתיבת הקלט, ומיישר את הכיוון בזמן אמת,
גם בזמן הזרמת התשובה.
- **שומר על אנגלית, קוד ונוסחאות בכיוון שמאל‑לימין:** בלוקי קוד, קוד מוטבע (inline),
נתיבי קבצים, פקודות, hashes, כתובות URL ונוסחאות LaTeX‏ (`$x^2$`, `$$...$$`) נשארים
קריאים. טבלאות עבריות נקראות מימין לשמאל.
- **מתקן את באג מסגרת החלון** במערכת Windows בשפת עברית (ראו ההערה למטה).
- **מגבה כל קובץ שהוא נוגע בו**, עם שחזור בלחיצה אחת וגלגול‑אחורה אוטומטי במקרה של
כשל.
- **נשאר מותקן גם אחרי עדכונים** (אופציונלי), באמצעות קיצור דרך או משימה מתוזמנת.

> **הערה על מסגרת החלון:** כש‑Windows מוגדר לעברית, Claude Desktop הופך את כל מסגרת
> החלון לכיוון RTL מעצמו. זהו באג קיים, לא נגרם על ידי התיקון. הוא דוחף את כפתורי
> החלון (מזעור / הגדלה / סגירה) מעל הכפתורים של Claude ומסתיר אותם. התיקון פותר זאת
> בכך שהוא מקבע את מסגרת החלון לכיוון LTR, בלי להשפיע על כיוון הטקסט של השיחה.

## התקנה מהירה

פתחו **Windows PowerShell** והריצו:

```powershell
irm https://raw.githubusercontent.com/CodesParadox/claude-desktop-hebrew-rtl/main/install.ps1 | iex
```

תופיע בקשת UAC — אשרו כדי להעניק הרשאות מנהל, ואז בחרו **1. התקנה** מהתפריט.

לחלופין, הורידו את הריפו, ומתוך התיקייה הריצו:

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1
```

## איך זה עובד (טכני)

Claude Desktop הוא אפליקציית Electron חתומה דיגיטלית, עם בדיקת תקינות על ארכיון
ה‑`app.asar`. כל שינוי בקוד ה‑UI שובר את הבדיקה הזו. הכלי עוקף זאת כך:

1. **הזרקה ל‑ASAR.** הכלי מחלץ את `app.asar` בעזרת `npx @electron/asar`, מזריק את
  מנוע זיהוי ה‑RTL ל‑renderer (ומקבע את מסגרת החלון ל‑LTR), אורז מחדש ומחליף את
   הארכיון. מנוע הזיהוי כתוב ב‑`[src/rtl-core.js](src/rtl-core.js)` (לוגיקה טהורה
   עם בדיקות יחידה) ו‑`[src/rtl-payload.js](src/rtl-payload.js)` (שכבת ה‑DOM);
   `tools/build-payload.ps1` מרכיב אותם לתוך `patch.ps1`.
2. **עקיפת התקינות — שיטה ראשית (fuse).** הכלי מכבה את ה‑fuse של Electron בשם
  `EnableEmbeddedAsarIntegrityValidation` באמצעות `@electron/fuses` על `claude.exe`.
   זוהי השיטה הפחות פולשנית, ואינה נוגעת ב‑`cowork-svc.exe` ולא במאגר השורש המהימן
   של Windows.
3. **גיבוי (fallback).** אם ה‑fuse נכשל, הכלי מחשב את ה‑hash החדש של כותרת ה‑ASAR
  ומחליף את ה‑hash המקורי בתוך `claude.exe` — אך ורק אם נמצאה התאמה יחידה וחד‑משמעית.
   אם ההתאמה מרובה או דו‑משמעית, הכלי עוצר ומבצע גלגול‑אחורה. גם נתיב זה אינו נוגע
   במאגר השורש המהימן.

כל קובץ שמשתנה מגובה קודם ל‑`*.bak` לצדו. בכל כשל מתבצע **גלגול‑אחורה אוטומטי**.

> מה הכלי **לא** עושה (בשונה מפרויקטים אחרים): הוא אינו מחליף תעודות חתימה בתוך
> `cowork-svc.exe`, ואינו מוסיף תעודה בחתימה‑עצמית למאגר השורש המהימן של Windows.

## אבטחה ואימות

`install.ps1` מאמת **חתימת RSA‑4096** על `patch.ps1` לפני שהוא מריץ אותו. ריפו שנפרץ
לבדו **אינו** מספיק כדי לשלוח קוד זדוני — התוקף יזדקק גם למפתח הפרטי הלא‑מקוון של
המתחזק. במקרה של כשל באימות, הכלי **נכשל סגור** (לא מריץ דבר).

**טביעת אצבע של המפתח הציבורי (SHA‑256):**

```
0e:50:21:7a:ca:c6:22:c1:04:91:13:f3:2c:39:f4:15:a1:cc:a5:4d:3a:49:e0:bf:7d:ae:5c:e8:91:05:13:d1
```

לאימות שחרור בלי להתקין:

```powershell
git clone https://github.com/CodesParadox/claude-desktop-hebrew-rtl
cd claude-desktop-hebrew-rtl
powershell -ExecutionPolicy Bypass -File tools\verify-signature.ps1
```

## הסרה / שחזור

הריצו את הכלי ובחרו **2. שחזור המצב המקורי**. זה משחזר את כל הקבצים המקוריים מהגיבוי,
מפעיל מחדש את ה‑fuse, ומוחק את סימון מצב התיקון. אם הפעלתם re‑patch אוטומטי, בחרו גם
**5** כדי להסיר את המשימה המתוזמנת.

## פתרון תקלות

- **"Node.js (npx) is required"**: התקינו Node.js מ‑[nodejs.org](https://nodejs.org/)
ופתחו מחדש את PowerShell.
- **התקנת חנות (MSIX) נדחתה**: התקינו את גרסת ה‑Squirrel‏ (`winget install Anthropic.Claude`).
- **Claude לא נפתח אחרי התיקון**: הריצו שוב, בחרו **2 (שחזור)**, ואז **1 (התקנה)**.
- **Claude התעדכן והתיקון נעלם**: הריצו את קיצור "עדכון מהיר", או השתמשו ב‑re‑patch
האוטומטי. עדכוני Claude דורסים את התיקון — זה צפוי.
